Quick Start

  1. Enter Book Information (title, subtitle, author).
  2. Create or paste a Book Brief.
  3. Choose structure mode: AutoBook, AI Sections or Manual Outline.
  4. Adjust settings in Book Size & Length and Writing Style & AI Control.
  5. Run Test Mode first, for example 2-3 chapters.
  6. Generate the full book when the test result looks correct.
Tip - Read through the guide before starting. It saves time and makes testing more consistent.

■System Requirements

BOLO is a Windows desktop application and requires Windows 10 or Windows 11.

You also need Microsoft Word or LibreOffice installed, plus your own OpenAI API key.

DOCX manuscripts are usually opened in Word or LibreOffice for review and editing after generation.

LibreOffice can be downloaded free of charge from the official website.

■1. Introduction

BOLO is designed for AI-assisted creation of books, especially nonfiction, guidebooks and research-based projects.

An experimental Fiction Mode also exists, but the system is primarily optimized for nonfiction workflows.

■2. When to Use BOLO

BOLO is usually used after the concept and research phase is complete. The most important preparation step is a clear and detailed Book Brief.

The software is then used to generate the outline, draft the manuscript, export files and produce the cover.

■3. Program Window

The program should normally be used in fullscreen mode. If the window is too small, some controls and buttons may not be visible.

■4. Recommended Workflow for Creating a Book

The most effective workflow usually starts with a detailed Book Brief. A longer and more specific brief generally gives better results.

A common workflow is:

1. Click Copy Base Prompt in BOLO.
2. Paste the prompt into an AI service such as ChatGPT.
3. Develop the book idea into a clear and detailed Book Brief.
4. Paste the finished brief into BOLO.
5. Choose a structure mode and generate the outline.
6. Run Test Mode before generating the full manuscript.
7. Review and edit the DOCX manuscript if needed.
8. Use Re-Export when you need a repaired or refreshed export after external editing.

For narrow subjects, be explicit about what each chapter should contain. For very broad subjects, keep expectations realistic in relation to the selected page count.

■5. Cost

A rough guideline is that a page of approximately 300 words costs around $0.01 to generate.

Fiction writing is usually 2-4 times more expensive than nonfiction because it requires more complex generation processes, more narrative control and more iterative prompting.

For this reason, it is recommended to use Test Mode before generating a full manuscript.

■6. Cover Generator - Recommended Workflow

Cover Generator creates a complete cover layout. Upload the DOCX manuscript, a front cover image and a back cover image.

The program calculates the spine automatically based on the manuscript length. You can then adjust Spine Position, title color, subtitle color, author color and spine color.

It is recommended to make a few test exports before final production.

Key settings
Book Size & Length, Writing Style & AI Control, Units and Anchor Data Density affect both the content and the final format of the book.
Good practice
Small changes in instruction strength, brief quality and Test Mode are often the fastest way to compare output quality before committing to a full run.

■7. Functions and Settings Explained

Book Information contains metadata such as title, subtitle and author.

Book Brief is the main instruction used by the AI and is the most important input in the workflow.

Copy Base Prompt copies a prompt template that can be used in ChatGPT or another AI tool to build a strong brief.

Save Setup stores your current settings so you can continue later without rebuilding the project.

AutoBook generates a complete outline automatically.

AI Sections creates subheadings when you already have chapter titles.

Manual Outline gives you full manual control of the structure.

Book Size & Length defines the number of volumes and pages per volume.

Writing Style & AI Control influences tone, style and how closely the AI follows the instructions.

Units switches between metric and imperial systems.

Anchor Data Density controls how many concrete facts, figures and data points appear in the text.

Test Mode generates a limited number of chapters for evaluation before a full run.

■8. Export and Re-Export

After a full generation, BOLO can produce DOCX, EPUB and, when needed, PRINT PDF.

DOCX is the main working format. EPUB is intended for e-book publishing. PRINT PDF is mainly used for print workflows and is therefore not always needed in the first step.

If you edit the manuscript in Microsoft Word, LibreOffice or another editor, the document structure may change. Save the file again as .docx and run Re-Export in BOLO to rebuild the internal structure and improve compatibility with publishing platforms such as Amazon KDP.

■9. Presentation

Generate Presentation is an additional tool and not the primary purpose of the software. It creates presentation material based on the Brief.

Presentation includes two modes:

Standard - generates a presentation directly from the Brief.

Website Overview - if you enter a website address in the field above, BOLO creates a summarized overview of the page and turns it into presentation material.

■10. Flyer

Generate Flyer creates a short informational sheet based on the Brief.

The flyer can be used for quick summaries, project descriptions or simple promotional material.
