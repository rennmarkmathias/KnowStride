■1. Introduction

BOLO is designed for AI-assisted creation of books, guidebooks, research-based projects, teaching material, illustrated manuscripts, covers and launch material.

A Fiction Mode is also available. It can write a first-draft manuscript for narrative projects, which can then be reviewed and, if necessary, edited.

BOLO is usually used after the concept and research phase is complete. The most important preparation step is a clear and detailed Book Brief.

The program should normally be used in fullscreen mode. If the window is too small, some controls and buttons may not be visible. You can also zoom the interface with Ctrl + and Ctrl - if needed.

■2. Recommended Workflow

The best results usually start with a detailed Book Brief. A longer and more specific brief gives BOLO clearer instructions about scope, audience, tone and must-cover content.

  1. Click Copy Base Prompt in BOLO.
  2. Paste the prompt into ChatGPT or another AI tool.
  3. Develop your idea into a clear Book Brief.
  4. Paste the finished brief into BOLO.
  5. Choose AutoBook, AI Sections or Manual Outline.
  6. Run Test Mode before generating a full manuscript.
  7. Generate the manuscript.
  8. Edit the DOCX manuscript if needed.
  9. Add illustrations if wanted.
  10. Go back to Generate Main and run Re-Export to create the final DOCX, EPUB and PRINT PDF files.

Note: The Base Prompt is only used to create your Book Brief. Do not paste the Base Prompt directly into BOLO as the final brief.

Test run advice: For a project you plan to publish, start with one to three chapters. Read at least one chapter yourself, and consider asking an AI chat to compare the test text with your Book Brief. This makes it easier to see whether the brief, outline, tone, length or settings should be adjusted before generating the full book.

If a narrow subject does not fill the target word count, expand the Book Brief with more detail and ask BOLO for more chapters — for example, increase an outline from 20 chapters to 25 or 30. A combination of both usually gives the best result.

■3. Cost

Text generation is usually low cost. A rough guideline is that one page of approximately 300 words costs about one cent to generate, depending on model choice and current API pricing.

Typical nonfiction projects often cost only a few dollars to draft. Fiction projects can cost more because they require deeper continuity, longer scenes and more revision.

Image generation is priced separately. If you plan an illustrated book, count both text generation and image generation.

Use Test Mode before a full run. Price comparison is available in Key / License and on the website.

■4. Illustrate

You can add illustrations after generating your manuscript. Regular images are the most stable option.

Basic workflow: Generate more suggestions than needed, keep the best ones, click Update marker DOCX, then click Generate Images. The updated DOCX becomes your new working source.

The Fastest Way

  1. Go to the Illustrate tab.
  2. Upload your manuscript.
  3. Choose Image style and Image size in DOCX.
  4. Set Number of suggestions.
  5. Click Suggest Image Placement.
  6. Review the suggestions and keep the best ones.
  7. Click Update marker DOCX.
  8. Click Generate Images.

Available fixed sizes are Extra small (handout), Small, Medium and Large. You can also let BOLO decide a mixed layout automatically.

Best practice: If you want 10 illustrations, generate 20–30 suggestions first and keep the strongest placements.

Manual Markers

You can write markers directly in your manuscript using normal parentheses. Use only IMAGE, DIAGRAM, TIMELINE, MAP and TABLE.

(IMAGE: A glacier moving through a mountain valley)

(IMAGE: M; A glacier moving through a mountain valley)

(DIAGRAM: Plate boundaries showing divergence, subduction, and transform motion)

(TIMELINE: Development of river deltas over time)

(MAP: Mountain ranges and river systems in Central Europe)

(TABLE: Climate patterns across regions)

Rule: format = (TYPE: description) or (TYPE: M; description). Size is optional: XS / S / M / L. If size is omitted, BOLO treats it as medium.

Own Images

The own-image section is at the bottom of the Illustrate window. You can upload your own images, add captions, and place matching markers such as (PICTURE 1) and (PICTURE 2) in the manuscript.

Important workflow note: Illustrations affect layout and total page count. Complete illustrations before creating the cover so the final page count and spine width are calculated correctly.

■5. Cover Generator

Cover Generator creates a complete wraparound cover layout. Upload the DOCX manuscript, a front cover image, a back cover image and, if wanted, a spine image.

BOLO calculates the spine width from the manuscript length. You can adjust Spine Position, title color, subtitle color, author color and spine color.

When creating cover images in ChatGPT or another AI tool, ask for 300 DPI and specify the trim size. For a spine image, also specify the page count, since spine width depends on the final manuscript length.

BOLO can adapt a reasonably sized spine image to the cover spine. If you upload front, back and spine images, the cover result is usually straightforward. Test exports are mainly useful when you write the spine manually with text, adjust manual positioning or want to check small visual details before final production.

The program can also generate cover text. You can use it as is or shorten it manually or with an AI tool.

If the cover is converted to PDF, do not open it immediately. Wait about 20 seconds for the file to be fully processed.

■6. Functions and Settings

Book Information contains metadata such as title, subtitle and author.

Book Brief is the main instruction used by the AI and is the most important input in the workflow.

AutoBook generates a complete outline automatically from your Book Brief.

AI Sections creates subheadings when you already have chapter titles.

Manual Outline gives you full manual control of chapters and subheadings.

Book Size & Length defines trim size, volumes and target length.

TOC contents controls the table of contents: choose Chapters only or Chapters + subheadings. The same choice is used during the first export and during Re-Export. The first export shows the selected table of contents structure, but page numbers are finalized through Re-Export. In the A4 School Handout format, the table of contents is placed on its own page between the cover page and the body text. For educational handouts, this page can also be used for brief teacher instructions or comments.

Writing Style & AI Control influences tone, style and how closely the AI follows instructions.

Units switches between metric, imperial and mixed unit handling.

Anchor Data Density controls how many concrete facts, figures and data points appear in the text.

Test Mode generates a limited test run before full production.

Save Setup stores your current settings so you can continue later.

■7. Export and Re-Export

After generation, BOLO exports DOCX, EPUB and PRINT PDF.

DOCX is the main working format for review and editing. EPUB is for e-book publishing. PRINT PDF is the final production format normally used for print.

If you edit the manuscript in Word, LibreOffice, OpenOffice or another editor, save the file again as .docx. Then go back to Generate Main and run Re-Export. BOLO rebuilds the document structure and creates fresh DOCX, EPUB and PRINT PDF export files.

Table of contents note: To get a numbered table of contents, run Re-Export. Normally you do this after all manuscript edits and any illustrations are finished. If you do not plan to change anything after generation, simply run Re-Export once to create the final numbered table of contents.

Practical Use and Testing

The best way to learn BOLO is through a few small test runs. Generate one to three chapters, read at least one chapter yourself, and let an AI chat compare the result with your Book Brief. This is especially useful before producing a complete book for publication.

As long as you keep your original generated manuscript file, you can safely create new versions and try different approaches.

System Feedback and Log

At the bottom of the program, the log window shows what the system is doing. If a button seems to do nothing, check the log first.

■8. Presentation and Flyer

Generate Presentation creates presentation material from the Brief. It includes Standard presentation and Website overview.

Generate Flyer creates a short informational sheet based on the Brief. It can be used for summaries, project descriptions or simple promotional material.

■5. Cover Generator

Cover Generator opretter et komplet wraparound-coverlayout. Upload DOCX-manuskriptet, et forsidebillede, et bagsidebillede og eventuelt et rygbillede.

BOLO beregner rygbredden ud fra manuskriptets længde. Du kan justere Spine Position, titelfarve, undertitelfarve, forfatterfarve og rygfarve.

Når du opretter coverbilleder i ChatGPT eller et andet AI-værktøj, så bed om 300 DPI og angiv trim size. For et rygbillede skal du også angive page count, da rygbredden afhænger af den endelige manuskriptlængde.

BOLO kan tilpasse et rygbillede i rimelig størrelse til coverryggen. Hvis du uploader forside-, bagside- og rygbilleder, er coverresultatet normalt ligetil. Testeksporter er især nyttige, når du skriver ryggen manuelt med tekst, justerer manuel placering eller vil kontrollere små visuelle detaljer før endelig produktion.

Programmet kan også generere covertekst. Du kan bruge den som den er eller forkorte den manuelt eller med et AI-værktøj.

Hvis coveret konverteres til PDF, skal du ikke åbne det med det samme. Vent cirka 20 sekunder, så filen er færdigbehandlet.

■7. Eksport og Re-Export

Efter generering eksporterer BOLO DOCX, EPUB og PRINT PDF.

DOCX er det vigtigste arbejdsformat til gennemgang og redigering. EPUB er til e-bogspublicering. PRINT PDF er det endelige produktionsformat, der normalt bruges til tryk.

Hvis du redigerer manuskriptet i Word, LibreOffice, OpenOffice eller en anden editor, skal du gemme filen igen som .docx. Gå derefter tilbage til Generate Main og kør Re-Export. BOLO genopbygger dokumentstrukturen og opretter friske DOCX-, EPUB- og PRINT PDF-eksportfiler.

Note om indholdsfortegnelsen: For at få en nummereret indholdsfortegnelse skal du køre Re-Export. Normalt gør du det efter alle manuskriptredigeringer og illustrationer er færdige. Hvis du ikke vil ændre noget efter generering, kan du blot køre Re-Export én gang for at oprette den endelige nummererede indholdsfortegnelse.

■7. Export och Re-Export

Efter generering exporterar BOLO DOCX, EPUB och PRINT PDF.

DOCX är det huvudsakliga arbetsformatet för granskning och redigering. EPUB används för e-bokspublicering. PRINT PDF är det slutliga produktionsformatet som normalt används för tryck.

Om du redigerar manuset i Word, LibreOffice, OpenOffice eller en annan editor, spara filen igen som .docx. Gå sedan tillbaka till Generate Main och kör Re-Export. BOLO bygger om dokumentstrukturen och skapar nya DOCX-, EPUB- och PRINT PDF-exportfiler.

Notering om innehållsförteckningen: För att få en numrerad innehållsförteckning, kör Re-Export. Normalt gör du detta efter att alla manusredigeringar och illustrationer är klara.
